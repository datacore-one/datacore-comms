## Trigger

`:AI:comms:calendar:` tag in org-mode tasks

## Purpose

This agent manages the content lifecycle from draft to posted, handles scheduling, and archives published content with metrics.

## Content Queue Structure

```
1-tracks/comms/campaigns/[quarter]/[month]/
├── calendar.md              # Monthly plan
├── drafts/                  # Generated, pending review
│   └── [date]-[platform]-[topic].md
├── approved/                # Human-approved, ready to post
│   └── [date]-[platform]-[topic].md
└── posted/                  # Published, archived with metrics
    └── [date]-[platform]-[topic].md
```

## Workflow States

```
DRAFT → APPROVED → POSTED

draft:
  - Generated by content-generator
  - Passed voice-enforcer
  - Awaiting human review

approved:
  - Reviewed in /today briefing
  - Moved by human or /approve-content
  - Ready for posting

posted:
  - Published to platform
  - Archived with metrics
  - Available for analytics
```

## Posting Workflow

```
1. Check approved/ folder for ready content
2. Determine optimal posting time
3. Connect to platform API (Late, Typefully)
4. Post content
5. Capture post URL/ID
6. Move to posted/ with metadata
7. Track initial engagement (if available)
```

## Web3 Channel Integration

### Farcaster (via Neynar API)

```python
# Example: Post cast to Farcaster
import requests

def post_to_farcaster(content, neynar_api_key, signer_uuid):
    """
    Post a cast to Farcaster via Neynar API.
    Max 320 characters per cast.
    """
    url = "https://api.neynar.com/v2/farcaster/cast"
    headers = {
        "accept": "application/json",
        "api_key": neynar_api_key,
        "content-type": "application/json"
    }
    payload = {
        "signer_uuid": signer_uuid,
        "text": content[:320],  # Enforce max length
        # Optional: embeds, channel_id, parent, etc.
    }
    response = requests.post(url, json=payload, headers=headers)
    return response.json()
```

**Neynar Setup:**
1. Sign up at neynar.com
2. Create API key
3. Create a signer (managed or self-hosted)
4. Store `NEYNAR_API_KEY` and signer UUID

**Channel Posting:**
```python
# Post to specific channel (e.g., /privacy)
payload = {
    "signer_uuid": signer_uuid,
    "text": content,
    "channel_id": "privacy"  # Channel to post to
}
```

### Telegram (Bot API)

```python
# Example: Post to Telegram channel
import requests

def post_to_telegram(content, bot_token, chat_id, parse_mode="Markdown"):
    """
    Post to Telegram channel via Bot API.
    Supports Markdown or HTML formatting.
    """
    url = f"https://api.telegram.org/bot{bot_token}/sendMessage"
    payload = {
        "chat_id": chat_id,  # @channelname or numeric ID
        "text": content,
        "parse_mode": parse_mode,
        "disable_web_page_preview": False
    }
    response = requests.post(url, json=payload)
    return response.json()
```

**Telegram Setup:**
1. Create bot via @BotFather
2. Add bot as admin to your channel
3. Get channel ID (numeric or @username)
4. Store `TELEGRAM_BOT_TOKEN` in env

**Media Posting:**
```python
# Post with image
url = f"https://api.telegram.org/bot{bot_token}/sendPhoto"
payload = {
    "chat_id": chat_id,
    "photo": image_url,
    "caption": content,
    "parse_mode": "Markdown"
}
```

### Discord (Webhook)

```python
# Example: Post to Discord channel via webhook
import requests

def post_to_discord(content, webhook_url, embed=None, username="Comms Bot"):
    """
    Post to Discord channel via webhook.
    Supports embeds for rich formatting.
    """
    payload = {
        "username": username,
        "content": content[:2000] if not embed else None,
    }

    if embed:
        payload["embeds"] = [embed]

    response = requests.post(webhook_url, json=payload)
    return response.status_code == 204  # Discord returns 204 on success
```

**Discord Embed Format:**
```python
embed = {
    "title": "MyProduct Update",
    "description": content,
    "color": 0x00D4AA,  # Brand color
    "fields": [
        {"name": "Type", "value": "Announcement", "inline": True},
        {"name": "Platform", "value": "MyProduct", "inline": True}
    ],
    "footer": {"text": "Your Organization"},
    "timestamp": "2026-01-26T12:00:00.000Z"
}
```

**Discord Setup:**
1. Server Settings → Integrations → Webhooks
2. Create webhook for target channel
3. Copy webhook URL
4. Store `DISCORD_WEBHOOK_URL` in env

## Cross-Platform Posting Strategy

When posting the same content across platforms:

1. **Start with X** - Highest reach, sets narrative
2. **Adapt for Farcaster** - Same core, crypto-native language
3. **Post to Telegram** - Add context, community tone
4. **Discord announcement** - Full details, embed format
5. **LinkedIn** (if appropriate) - Professional framing

Use the **web3-adapter** agent to convert content between platforms.

## Error Handling

1. **API failure**: Retry 3 times, then mark content as "failed" with error

2. **Rate limit**: Queue for next available slot

3. **Missing credentials**: Return setup instructions, don't attempt post

4. **Content too long**: Flag for revision, don't truncate automatically
